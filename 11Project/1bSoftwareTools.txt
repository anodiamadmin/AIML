We will use MS 365 Teams as our primary software for group meetings, with WhatsApp (Other) as a supplementary real-time communication tool. MS Teams will be used for all formal meetings, including weekly team discussions and sessions with lecturers. It offers video conferencing, screen sharing, recording, and chat. We will use screen sharing to review code, discuss model outputs, and present findings. Recordings will help absent members stay aligned. WhatsApp will be used only for quick, informal communication such as scheduling, reminders, and minor clarifications, not for formal meetings. This choice ensures structured collaboration. MS Teams is preferred over tools like Zoom or Slack due to its integration with the UNSW ecosystem and its comprehensive meeting features. WhatsApp complements this by enabling fast coordination. For example, meetings will be on Teams, while day-to-day updates occur via WhatsApp.


We will use GitHub, Trello and Google Drive to manage our work collaboratively. GitHub will be the main platform for version control, storing code, datasets and documentation. We will use repositories, branching and pull requests to manage parallel development and ensure reproducibility. Trello will be used for task management using a Scrum-board, with tasks organised into To Do, In Progress and Completed to track progress and assign responsibilities. Google Drive will be used for file sharing and collaborative editing of reports and documents. These tools were chosen for their complementary strengths. GitHub supports structured and reproducible development, Trello enables clear tracking of tasks, and Google Drive allows efficient document collaboration. Compared to alternatives, they are widely used, easy to integrate and suitable for both technical and non-technical work. For example, tasks in Trello will link to GitHub work, while reports will be developed in Google Drive.


We will use Python and R/RStudio for data analysis. Python will be the primary tool due to its strong ecosystem for data science. We will use pandas and numpy for data processing, matplotlib and seaborn for visualisation, scikit-learn for machine learning models such as regression and XGBoost, and PyTorch for LSTM models. Jupyter notebooks will support interactive analysis and reproducibility. R/RStudio will be used for statistical modelling and time series analysis, particularly ARIMA and exploratory analysis, using packages such as forecast and ggplot2. These tools were chosen for their complementary strengths. Python provides flexibility and scalability for machine learning, while R offers strong statistical capabilities. Compared to alternatives like MATLAB or SAS, they are open source, widely used and better suited for collaborative academic work. For example, data will be cleaned and modelled in Python, while time series validation and visualisation can be performed in R.


We will use Jupyter notebooks, Google Colab and RMarkdown to write our report in a reproducible manner, with MS Word used only for final formatting and submission. Jupyter and Colab will be used to combine Python code, outputs and explanations, enabling clear presentation of data processing, modelling and visualisation. Colab also allows cloud-based execution and easy sharing across the team. RMarkdown will integrate R code with text and generate structured reports. These tools support embedding code and results directly, ensuring reproducibility. They were chosen because they align with our use of Python and R and support collaboration. Compared to LaTeX or Overleaf, they provide better integration with code execution. For example, results and plots can be generated and updated automatically, then exported to Word for final presentation.


We will use PowerPoint and Zoom for our oral presentation. PowerPoint will be used to design and deliver the presentation slides. It allows clear structuring of content, visualisation of results and integration of charts and images. We will use it to present key findings, models and insights in a concise and professional format. Zoom will be used to record and deliver the presentation, enabling screen sharing and video recording. This ensures smooth delivery and allows us to capture the presentation for submission. These tools were chosen due to their ease of use, familiarity within the team and strong compatibility with university requirements. Compared to alternatives like Prezi or Beamer, PowerPoint provides better control over slide design and is more widely accepted. Zoom offers reliable recording features without additional setup. For example, slides will be created in PowerPoint and presented and recorded via Zoom for submission.


We will also use Visual Studio Code as an additional tool. Visual Studio Code will serve as a lightweight code editor for developing and managing Python scripts, with features such as extensions, debugging and GitHub integration. It supports efficient coding, version control and project organisation. This tool was selected because it is flexible, widely used and integrates well with our existing workflow. Compared to other editors, it provides a simple interface with powerful features for development. For example, code can be written and tested in Visual Studio Code before being integrated into Jupyter notebooks or pushed to GitHub for collaboration.
